# АИС учёта кадров и клиентов · ГБУК «Дом Науки и Техники»

> Веб-приложение для автоматизации учёта сотрудников, повышения квалификации, клиентов и договоров обучения.

Система разработана для ГБУК «Дом Науки и Техники» как замена ручному учёту в Excel и бумажных журналах. Основной интерфейс — административная панель на Filament v4 с дашбордом, экспортом, ролевым доступом и журналом действий.

## Возможности

### Учёт кадров
- Карточки сотрудников: ФИО, должность, подразделение, дата приёма, образование, контакты.
- Повышение квалификации с автоматическим контролем срока (3 года по ст. 47 ФЗ № 273-ФЗ).
- Прикрепление сканов дипломов и удостоверений (PDF, JPG, PNG до 10 МБ).
- Виджет на дашборде — кто должен пройти повышение в ближайшие 90 дней.

### Учёт клиентов и договоров
- База физических и юридических лиц с разными обязательными полями.
- Договоры на обучение: статус (Ожидает оплаты / Оплачен / Обучается / Завершён), сумма, курс, клиент.
- **Печать договора в Word** — генерация `.docx` по шаблону `storage/app/templates/contract.docx`.
- Прикрепление скана подписанного договора.

### Справочники
- Подразделения, должности, курсы обучения.

### Дашборд
- Пять плиток ключевых показателей (сотрудники, клиенты, договоры, требующие повышения квалификации, ожидающие оплаты).
- Линейный график заключения договоров за 12 месяцев.
- Столбиковая диаграмма финансового среза по статусам.
- Кольцевая диаграмма распределения штата по подразделениям.
- Таблица контроля сроков повышения квалификации.

### Администрирование
- Две роли: **Администратор** и **Сотрудник** (через `App\Enums\UserRole`).
- Журнал действий — кто, когда и какую запись создал / изменил / удалил.
- Глобальный поиск (`Ctrl/Cmd + K`) по сотрудникам, клиентам, договорам, курсам, подразделениям.
- Экспорт списков в **XLSX** и **CSV**.
- Светлая и тёмная темы.

## Установка

### Требования

- PHP 8.3+ с расширениями `pdo_mysql`, `mbstring`, `gd`, `zip`, `intl`
- Composer 2
- Node.js 20+ и npm
- MySQL 8 (или MariaDB 10.6+) — для прода
- Веб-сервер (nginx / Apache) или OSPanel / Laragon / Herd для локальной разработки

### Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone https://github.com/<your-org>/ais-dnit.git
cd ais-dnit

# 2. Зависимости
composer install
npm install

# 3. Окружение
cp .env.example .env
php artisan key:generate

# 4. Настроить БД в .env (DB_DATABASE, DB_USERNAME, DB_PASSWORD)
# и создать пустую базу:
#   mysql -uroot -e "CREATE DATABASE ais_dnit CHARACTER SET utf8mb4;"

# 5. Миграции и тестовые данные
php artisan migrate --seed

# 6. Символическая ссылка для загруженных файлов
php artisan storage:link

# 7. Сборка фронтенда
npm run build
```

### Запуск в режиме разработки

```bash
composer run dev
```

Команда параллельно запускает сервер Laravel, очередь и Vite в режиме HMR.

Приложение откроется на `http://localhost:8000`, корневой URL автоматически редиректит на `/admin`.

### Учётные записи по умолчанию (после `--seed`)

| Роль          | E-mail              | Пароль     |
|---------------|---------------------|------------|
| Администратор | `admin@dnit.local`  | `password` |
| Сотрудник     | `staff@dnit.local`  | `password` |

**После первого входа смените пароли в разделе Администрирование → Пользователи.**

## Структура проекта

```
app/
├── Enums/             # UserRole, ClientType, ContractStatus
├── Filament/
│   ├── Resources/     # CRUD для всех сущностей
│   ├── Widgets/       # Графики и таблицы дашборда
│   ├── Concerns/      # Переиспользуемые трейты
│   └── Exports/       # Экспорт в XLSX / CSV
├── Models/            # Eloquent: Employee, Client, Contract, …
├── Policies/          # Политики доступа по ролям
└── Services/
    └── ContractDocumentGenerator.php  # Генерация договоров .docx

database/
├── migrations/        # Схема БД
├── seeders/           # Начальные данные и тестовые записи
└── factories/

docs/
└── USER_GUIDE.md      # Руководство для конечных пользователей (HR, бухгалтерия)

storage/app/templates/
└── contract.docx      # Шаблон договора с плейсхолдерами
```
